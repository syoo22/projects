# TIL 제목 & 해시태그 분류 Apps Script 활용 가이드

## 1. 문서 목적

본 문서는 TIL 챌린지 참여자의 블로그 제출 링크를 검수하여  
커피쿠폰 지급 대상자를 판별하는 Google Apps Script 자동화 시스템의  
구조 및 운영 방법을 정리한 인수인계용 가이드입니다.

업무 담당자가 변경되더라도 동일한 기준으로 운영할 수 있도록  
검수 기준 및 플랫폼별 예외 사항을 포함하여 작성되었습니다.

---

## 2. 도입 배경

기존 방식은 300건이 넘는 블로그 URL을 수동으로 접속하여  
제목 및 해시태그에 특정 키워드(‘멋쟁이사자처럼’)가 포함되어 있는지 확인 후  
직접 체크하는 구조였습니다.

해당 작업은 월 2~3시간 이상 소요되는 반복 업무였으며,  
이를 개선하기 위해 Google Apps Script 기반 자동 검수 시스템을 구축하였습니다.

---

## 3. 자동화 동작 원리

1. 스프레드시트 A열의 블로그 URL을 순차적으로 읽음
2. 해당 URL에 접속하여 HTML 구조를 분석
3. 플랫폼별 제목 및 해시태그 영역을 추출
4. ‘멋쟁이사자처럼’ 키워드 포함 여부 확인
5. 결과를 다음과 같이 표기

| 열 | 기준 |
|----|------|
| F열 | 제목에 키워드 포함 시 O, 미포함 시 X |
| G열 | 해시태그에 키워드 포함 시 O, 미포함 시 X |
| H열 | F, G 모두 O일 경우 O, 하나라도 X일 경우 X |

---

## 4. 스프레드시트 구조

| 열 | 내용 |
|----|------|
| A | 블로그 URL |
| F | 제목 키워드 포함 여부 |
| G | 해시태그 키워드 포함 여부 |
| H | 커피쿠폰 지급 대상 여부 |

※ A열 외 다른 열은 자동으로 입력됨

---

## 5. 플랫폼별 검수 기준 및 예외 사항

### 1) 네이버 블로그

- 제목은 HTML 구조상 자동 추출 가능
- 태그는 일부 게시글에서 HTML 내 노출 구조가 불규칙함
- 일부 경우 태그 영역이 스크립트로 렌더링되어 자동 검수가 정확하지 않을 수 있음

⚠️ 따라서 네이버 블로그의 태그는
자동 결과 확인 후 필요 시 수동 검수 보완이 필요함

---

### 2) 벨로그 (Velog)

- 제목 및 태그가 비교적 명확한 클래스 구조로 제공됨
- 자동 검수 정확도 높음
- 별도 수동 보완 필요 없음

---

### 3) 티스토리

- HTML 구조 기반으로 제목 및 태그 추출 가능
- 블로그 스킨에 따라 구조 차이가 있을 수 있음
- 오류 발생 시 해당 URL 개별 확인 필요

---

### 4) 노션 페이지

- 노션은 일반 블로그 플랫폼과 달리
  전통적인 “태그 기능”이 존재하지 않음
- 태그 개념이 명확히 분리되어 있지 않아
  시스템상 해시태그 영역 추출이 불가

따라서:

- 제목은 검수 가능
- 해시태그는 전부 X로 표기됨

※ 노션 제출자는 운영 정책상 별도 기준 적용 여부 검토 필요

---

## 6. 운영 시 유의사항

1. 키워드 변경 시 코드 내부 검색 문자열 수정 필요
2. HTML 구조 변경 시 플랫폼별 선택자 수정 필요
3. 대량 요청 시 일시적 차단 가능성 있음 (재실행으로 해결 가능)
4. 네이버 태그는 수동 보완 가능성 항상 염두

---

## 7. 실행 방법

1. Google Spreadsheet 열기
2. 확장 프로그램 → Apps Script 실행
3. 스크립트 실행 버튼 클릭
4. 결과 F/G/H열 자동 반영 확인

---

## 8. 기대 효과

- 월 2~3시간 이상 반복 업무 절감
- 검수 기준 표준화
- 인수인계 리스크 감소
- 운영 자동화 기반 마련

---

## 9. 향후 개선 가능 사항

- 네이버 태그 정확도 개선 로직 추가
- 오류 URL 자동 로그 기록 기능
- UI 버튼 생성으로 실행 편의성 개선
