# TIL 제목 & 해시태그 분류 Apps Script 활용 가이드

## 1. 문서 목적

본 문서는 TIL 챌린지 참여자의 블로그 제출 링크를 검수하여  
경품 지급 대상자를 판별하는 Google Apps Script 자동화 시스템의  
구조 및 운영 방법을 정리한 인수인계용 가이드입니다.

업무 담당자가 변경되더라도 동일한 기준으로 운영할 수 있도록  
검수 기준 및 플랫폼별 예외 사항을 포함하여 작성되었습니다.

---

## 2. 도입 배경

기존 방식은 300건이 넘는 블로그 URL을 수동으로 접속하여  
제목 및 해시태그에 특정 키워드(‘멋쟁이사자처럼’)가 포함되어 있는지 확인 후  
직접 체크하는 구조였습니다.

해당 작업은 월 2~3시간 이상 소요되는 반복 업무였으며,  
이를 개선하기 위해 Google Apps Script 기반 자동 검수 시스템을 구축하였습니다.

---

## 3. 시스템 개요

본 시스템은 네이버 블로그, 티스토리, Velog 등 다양한 플랫폼의 블로그 URL HTML 구조를 분석하여 제목 및 해시태그에 "멋쟁이사자처럼" 키워드 포함 여부를 자동 검증하고, 결과를 스프레드시트에 표시하는 Google Apps Script 기반 자동 검증 시스템입니다.

플랫폼마다 HTML 구조가 다르기 때문에 각 플랫폼별로 제목 및 해시태그 영역을 구분하여 추출하도록 설계되었습니다.

---

## 4. 자동화 동작 원리

1. 스프레드시트 A열의 블로그 URL을 순차적으로 읽음
2. 해당 URL에 접속하여 HTML 구조를 분석
3. 플랫폼별 제목 및 해시태그 영역을 추출
4. ‘멋쟁이사자처럼’ 키워드 포함 여부 확인
5. 결과를 다음과 같이 표기

| 열 | 기준 |
|----|------|
| F열 | 제목에 키워드 포함 시 O, 미포함 시 X |
| G열 | 해시태그에 키워드 포함 시 O, 미포함 시 X |
| H열 | F, G 모두 O일 경우 O, 하나라도 X일 경우 X |

---

## 5. 스프레드시트 구조

| 열 | 내용 |
|----|------|
| A | 블로그 URL |
| F | 제목 키워드 포함 여부 |
| G | 해시태그 키워드 포함 여부 |
| H | 경품 지급 대상 여부 |

※ A열 외 다른 열은 자동으로 입력됩니다.

---

## 6. 플랫폼별 검수 기준 및 예외 사항

### 1) 네이버 블로그

- 제목은 HTML 구조상 자동 추출 가능
- 해시태그 영역의 HTML 구조가 일정하지 않으며 일부 게시글에서는 태그 구조가 다르게 표시될 수 있음
- 일부 경우 태그 정보가 페이지 로딩 이후 스크립트로 동적으로 렌더링되어 Apps Script에서 HTML을 가져올 때 태그 정보가 포함되지 않는 문제가 발생함

초기 자동화 개발 과정에서는 모든 플랫폼에 동일한 방식의 태그 추출 로직을 적용하였으나, 네이버 블로그의 경우 화면에서는 해시태그가 정상적으로 표시됨에도 불구하고 Apps Script에서 가져온 HTML에는 태그 정보가 포함되지 않는 사례가 확인되었습니다.

이는 네이버 블로그가 일부 경우 태그 정보를 자바스크립트로 동적으로 렌더링하거나 자동 요청(UrlFetchApp)에 대해 제한된 HTML을 반환하기 때문으로 판단됩니다.

따라서 네이버 블로그의 태그 자동 추출이 실패하는 경우를 구분하기 위해 네이버 블로그 URL이면서 태그가 추출되지 않았을 경우 스프레드시트 I열에 `태그 자동추출 불가` 로그가 표시되도록 구현하였습니다.

이 로그가 표시된 경우에는 태그 자동 검수가 불가능한 상태이므로 운영자가 해당 블로그에 직접 접속하여 해시태그를 확인한 후 G열 값을 수동으로 수정해야 합니다.

⚠️ 따라서 네이버 블로그 제출 건은 자동 검수 결과 확인 후 필요 시 태그를 수동으로 보완하여 검수를 진행합니다.

---

### 2) 벨로그 (Velog)

- 제목 및 태그가 비교적 명확한 클래스 구조로 제공됨
- 자동 검수 정확도 높음
- 별도 수동 보완 필요 없음

---

### 3) 티스토리

HTML 구조 기반으로 제목 및 태그 추출 가능

블로그 스킨(테마)마다 태그 영역 클래스/구조가 달라 동일 로직으로는 오탐 또는 미검출이 발생할 수 있음

기존 방식처럼 페이지 전체에서 /tag/ 링크를 검색하면 글에 태그가 없어도 사이드바 또는 태그 클라우드의 태그를 게시글 태그로 오인하여 G열이 O로 표시되는 문제가 발생할 수 있습니다.


예: 게시글 태그가 비어 있어도 사이드바 태그 목록에 멋쟁이사자처럼이 있으면 O 처리되는 오류 발

이를 해결하기 위해 태그가 실제로 포함된 컨테이너 영역(scope) 내부에서만 태그를 추출하도록 수정하였습니다.

적용한 대표 컨테이너 예시: entry-tag, tags, tag_content, post-tags, article-tag, items, box-tag, #tags

오류 또는 미검출 발생 시 해당 URL을 개별 확인한 후 해당 스킨의 태그 컨테이너 HTML 구조(클래스 또는 ID)를 코드에 추가 등록해야 합니다.

---

### 4) 노션 페이지

- 노션은 일반 블로그 플랫폼과 달리
  전통적인 “태그 기능”이 존재하지 않음
- 태그 개념이 명확히 분리되어 있지 않아
  시스템상 해시태그 영역 추출이 불가

따라서:

- 제목은 검수 가능
- 해시태그는 전부 X로 표기됨

※ 노션 제출자는 운영 정책상 별도 기준 적용 여부 검토 필요

---

## 7. 운영 시 유의사항

1. 키워드 변경 시 코드 내부 검색 문자열 수정 필요
2. HTML 구조 변경 시 플랫폼별 선택자 수정 필요
3. 대량 요청 시 일시적 차단 가능성 있음 (재실행으로 해결 가능)
4. 네이버 태그는 수동 보완 가능성 항상 염두

---

## 8. 실행 방법

1. Google Spreadsheet 열기
2. 확장 프로그램 → Apps Script 실행
3. 스크립트 실행 버튼 클릭
4. 결과 F/G/H열 자동 반영 확인

---

## 9. 기대 효과

- 월 2~3시간 이상 반복 업무 절감
- 검수 기준 표준화
- 인수인계 리스크 감소
- 운영 자동화 기반 마련
